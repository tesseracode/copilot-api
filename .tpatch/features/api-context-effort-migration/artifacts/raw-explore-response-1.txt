- Remove suffix normalization logic that converts `max` → `xhigh` or creates `-1m` variants for Claude 4.6+
- Keep [1m] suffix stripping as no-op (don't resolve to variant)
- Update `anthropicToCopilotModelId()` to fall back to base model when variant not in catalog (already works per analysis)
- Add helper: `getModelCapabilities(modelId)` → returns model metadata including `reasoning_effort` array
- Add helper: `getValidEffortValues(modelId)` → returns supported effort values from capabilities

## 2. **src/lib/api-config.ts**
- Refactor `supportsEffort(modelId)` to check `capabilities.supports.reasoning_effort` array instead of `-1m` suffix in ID
- Add `isValidEffort(modelId, effortValue)` → validates effort against model's supported array
